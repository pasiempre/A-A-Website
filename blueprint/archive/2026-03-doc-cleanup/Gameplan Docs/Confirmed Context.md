## 3. Confirmed Product Direction (Most Important)

### 3.1 Resource/Training Library

- Building out a full employee resource/training library is not realistic right now due time constraints.
- The SOP knowledge is still valuable and should be captured later, especially for scaling and quality reinforcement.
- Decision: defer deep resource-library content creation for now.

### 3.2 Photo and QA Workflow

- Photo capture is not a reliable final-quality verification method for fine detail (dust/streaks may not show clearly).
- Crew photo use should focus on issue documentation, not full-quality signoff.
- Issue photo examples:
  - Paint in tub
  - Damage/hole in wall
  - Broken items
  - Mess/out-of-place conditions
- Each employee should be able to report issues themselves from their own job workflow.
- Final quality review point is Areli, not an on-site QC role.
- Rework is a real workflow: if something is not done properly, Areli sends work back and shares photo evidence of what still needs correction.
- Desired QA workflow addition: Areli should be able to open a work card in review mode, attach photos, and list specific correction items.
- Work card creation is often repetitive; template-based cards and duplication are important.

### 3.3 Work Card System

- A work-card style system is the right core model.
- Work cards should include at minimum:
  - Project/job name
  - Clean type (rough clean, final clean, touch-up, first final, second final as needed)
  - Address/location
  - Specific detail list/tasks
  - Expected duration/time
  - Issue reporting action directly on the card
